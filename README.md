### 배포 링크

### 이 앱을 만든 이유
- 어떤 문제/불편함을 해결하려고 했나요?
  - **잔액 중심의 생활비 관리:** 기존 가계부 앱들은 '지출 총액' 확인에 집중되어 있어, 정작 중요한 **'설정한 예산 대비 남은 금액'**을 한눈에 파악하기 어려웠습니다. 자취생에게 꼭 필요한 **'쓸 수 있는 돈'**을 직관적으로 확인하기 위해 개발했습니다.
   
  - **루틴의 통합 관리:** 하루에 대한 기록을 하는 가계부와 투 두 앱은 서로 비슷한 성질을 가졌다고 생각합니다. 하지만 서로 다른 앱에서 기록해야한다는 번거로움이 있습니다. 지출 기록과 일정 관리를 하나의 앱에서 해결함으로써, 기록의 번거로움을 줄이고 직관적인 확인이 가능하도록 했습니다.
  >
  > 
- 이 앱을 언제, 어떻게 사용할 건가요?
  - **하루의 마무리:** 잠들기 전, 오늘 쓴 비용을 기록하며 남은 생활비를 체크하고 동시에 내일 해야 할 일을 메모합니다.

> "오늘 내가 얼마를 썼는지보다, **앞으로 얼마를 더 쓸 수 있는지**가 더 궁금했습니다." 처음 자취를 시작하며 마주한 고민이었습니다. 시중의 가계부들은 단순히 소비를 나열할 뿐, 내 예산 안에서 살고 있는지 알려주지 않았습니다. 그래서 **'남은 금액'에 집중한 가독성 좋은 가계부**를 직접 만들게 되었습니다.
> 여기에 한 가지를 더 더했습니다. 매일 밤 지출을 정리하다 보면 자연스럽게 내일의 일정을 고민하게 됩니다. '가계부'와 '할 일 관리'를 한 곳에 담아, 사소한 기록들이 모여 건강한 경제 습관과 계획적인 일상을 만들 수 있도록 설계했습니다.


### 주요 기능
- 한 달 예산을 설정합니다. 
  - 특정 달에 추가 예산이 생길 경우 해당 달만 예산을 추가할 수 있습니다.
- 지출을 기록합니다.
  - 달력에 해당 일자의 총 소비 금액이 표시됩니다.
  - 앱 상단에 남은 생활비가 표시됩니다.
- 할 일을 기록합니다.
  - 할 일의 색깔을 지정하여 카테고리를 나눌 수 있습니다.
- 간단한 메모를 날짜별로 기록합니다.

[데일리 뭐하지?](https://gemini.google.com/share/30ecb864e907)

### 이 앱을 만든 이유
- 어떤 문제/불편함을 해결하려고 했나요?
  - 데일리 미팅 주제를 정하는 데 많은 시간이 소비되는 문제점을 해결하고자 했습니다.
- 이 앱을 언제, 어떻게 사용할 건가요?
  - 데일리 미팅 시작 전 크루가 모두 모였을 때 사용합니다.
  - 크루들과 마음에 드는 선택지를 고르고, 그 결과로 도출된 주제를 가지고 데일리 미팅을 진행합니다.

### 주요 기능
- 데일리 미팅 인원과 진행 시간에 적합한 주제를 추천합니다.

- 마음에 드는 주제를 캘린더에 저장할 수 있습니다.

### AI 기능
- 어떤 AI 기능을 추가했나요?
  - 사용자가 입력한 인원, 시간, 그리고 3단계의 선택 결과를 바탕으로 최적의 미팅 주제를 생성하여 추천합니다.
- AI 기능이 앱에서 어떤 역할을 하나요?
  - 앱의 핵심 기능인 데일리 미팅 주제를 추천합니다.
 
### 릴레이 회고
- 페어 프롬프트 릴레이를 진행하기 전까지는 5분이 꽤 길다고 생각했습니다. 하지만 실제로 진행해 보니 대개 한 번의 프롬프트를 작성하고 결과물을 확인하는 것만으로도 5분이 지나갔습니다.
- 페어와 함께 진행하니 혼자일 때보다 훨씬 다양한 아이디어가 나왔습니다. 혼자라면 생각지 못했을 방식을 서비스에 도입할 수 있었고, 의견이 다를 때는 각자의 프롬프트를 모두 실행해보며 더 나은 방향을 찾아갔습니다. 덕분에 결과물의 완성도가 훨씬 높아졌습니다.

## AI 사용 일지

### 1. 초기 프롬프트
```
생활비 관리 앱을 만들거야. 목표 사용자는 자취를 하는 사회 초년생이나 대학생이야. 이 웹앱의 핵심 기능은 한달 목표 생활비를 지정하고, 소비 내역을 직접 입력하며, 달력 ui를 통해 시각적으로 소비 내역을 확인하고, 현재 남은 생활비를 확인하며 관리하는 것이야. 
HTML,CSS, JavaScript만 사용해서 만들어줘. 다른 프레임워크는 사용하지마.
```

### 2. 프롬프트 개선 과정
- 처음 결과물의 문제점은 무엇이었나요?
  - 큰 틀의 기능은 구현되었지만, 사용자를 배려한 세부 기능들이 부족했습니다.
    - 예: 생활비 추가 기능, 달력 클릭 시 지출 상세 확인, 지출 수정 등
  - 지출 내역이 늘어날 때 페이징이나 스크롤이 적용되지 않고 화면에 그대로 전부 나열되었습니다.
- 어떤 식으로 프롬프트를 수정했나요?
  - 사용자 편의를 위한 세부 기능들을 직접 언급하여 개발하도록 요청했습니다.
  - 지출 내역 영역에 기본적으로 2개만 보이게 하고, 나머지는 스크롤로 확인하도록 구체적으로 지정했습니다.
- 몇 번의 반복을 거쳤나요?
  - 13번의 프롬프트를 작성하였습니다.

### 3. 효과적이었던 프롬프팅 전략
- 어떤 방식으로 요청했을 때 원하는 결과가 나왔나요?
  - 금지 해야하는 부분을 명확히 명시해주고, 
  - 추가 및 수정 사항을 구체적으로 설명했을 때 원하는 결과가 나왔습니다.
  - 특히 디자인의 경우, 특정 요소의 위치와 스타일을 상세히 지정해 주어야 의도한 대로 구현되었습니다.
- 반대로 효과가 없었던 방식은 무엇이었나요?
  - 디자인 부분에 있어, 직접 어떠한 디자인을 생성 혹은 그려달라고 할 경우 기대한 수준의 결과물을 받지 못했습니다.

### 4. 배운 점
- Gemini Canvas를 효과적으로 사용하기 위한 나만의 팁이 있다면?
  - 처음 프롬프트를 작성하기 전 서비스의 전반적인 개요와 틀을 미리 잡아두고, 기능의 큰 틀부터 요청하면 좋은 것 같습니다.
  - 만약 중간에 기능이 추가된다면 어느 위치에 어떤 식으로 기능을 추가하는지 명시해줘야 원하는 결과가 나왔습니다.
